## 컴포넌트별 역할

| 컴포넌트 | 기술 | 역할 |
|---|---|---|
| Web 클라이언트 | Next.js 15 (App Router) | SSR/SSG 피드·카드 상세 페이지, SEO |
| 모바일 앱 | React Native (Expo) | iOS·Android 네이티브 앱 |
| API 서버 | FastAPI (Python 3.12) | REST API, 인증, 피드, 추천, 검색 |
| 배치 워커 | Celery | 수집·요약·번역·저장 비동기 태스크 |
| 스케줄러 | Celery Beat | KST 0/6/12/18시 배치 트리거 |
| 메인 DB | PostgreSQL (RDS / Neon) | 카드·사용자·로그 영구 저장 |
| 캐시 / 큐 | Redis (Upstash / ElastiCache) | 피드 캐시, 추천 캐시, Celery 브로커 |
| 검색 엔진 | Meilisearch → Elasticsearch | 한국어 태그·카테고리 복합 검색 |
| AI 처리 | Claude API + LangChain | 요약·번역·분류 프롬프트 체이닝 |
| 번역 검증 | sentence-transformers (로컬) | 역번역 코사인 유사도 계산 (비용 0) |
| 모니터링 | Sentry + Grafana + Prometheus | 에러 추적·배치 상태·API 성능 |
| CI/CD | GitHub Actions | PR 자동 테스트 + ECR 푸시 + 배포 |
| 배포 | AWS ECS Fargate (or Railway) | API 서버·배치 워커 독립 스케일링 |

## 배포 구성

```
ECS Cluster
├── api-service         (FastAPI, 상시 가동, 수평 확장)
├── batch-worker        (Celery Worker, 배치 시간대 스케일업)
└── beat-scheduler      (Celery Beat, 1 task, 상시 가동)

RDS PostgreSQL          (Multi-AZ, 자동 백업)
Upstash Redis           (Serverless, 비용 최적화)
Meilisearch             (단일 인스턴스, MVP)
```

## 데이터 흐름 (요청 경로)

```
사용자 → CDN → Next.js(SSR) → FastAPI → Redis 캐시 히트
                                       → PostgreSQL (캐시 미스)
                                       → Meilisearch (검색)
```
